Mini Incident Response Plan
What this is
This repository contains a Mini Incident Response Plan written for a hypothetical
breach scenario at a fictional company, ZenKart Commerce.
Scenario: A Customer Support employee opened a malicious email attachment
disguised as a customer complaint. The attachment installed credential-stealing
malware and tricked the employee into re-entering their password on a fake login
prompt. The attacker used this access to deploy ransomware, encrypting files across
multiple internal servers and shared drives.
The plan follows the NIST Incident Response Lifecycle (NIST SP 800-61):
Preparation
Detection & Analysis
Containment
Eradication
Recovery
Lessons Learned (Post-Incident Review)
It also includes an internal communication plan (who is notified, and roughly
when) and three preventative measures tied directly to the root cause of the
breach.
Files
`Incident_Response_Plan.pdf` — the full incident response plan report.
How to "run" it
This is a written report, not code — there's nothing to execute. Simply open
`Incident_Response_Plan.pdf` in any PDF viewer to read it.
